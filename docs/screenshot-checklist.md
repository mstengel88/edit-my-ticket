# Screenshot Checklist

Use this checklist to capture App Store screenshots that match the real workflows in this project.

## General capture rules

- Use realistic but non-sensitive demo data.
- Keep dates, totals, names, and products legible.
- Avoid empty states unless you intentionally want one.
- Use the native iOS build in Xcode, not the browser.
- Capture in portrait first, then add iPad landscape if it tells the story better.

## iPhone screenshot set

### 1. Ticket list

Show:

- A populated ticket list
- Visible status badges
- At least one completed or pending ticket

Best screen:

- Main Tickets tab in list mode

### 2. Ticket editor

Show:

- Customer, product, truck, and amount fields
- Save, Print, and Email actions visible

Best screen:

- Edit Ticket view with a filled-in ticket

### 3. Ticket preview

Show:

- Printable ticket layout
- Clear branding and ticket structure

Best screen:

- Ticket preview after selecting a filled-in ticket

### 4. Reports

Show:

- Summary totals
- Filters and report sections

Best screen:

- Reports page with real ticket data for a useful date range

### 5. Customer or product management

Show:

- Searchable record management
- Add/edit workflow context

Best screen:

- Customers page or Products page with a populated table

## iPad screenshot set

### 1. Ticket list grid

Show:

- The improved two-column tablet ticket layout
- Search and multiple populated cards

Best screen:

- Tickets page on iPad portrait in list mode

### 2. Ticket editor on iPad

Show:

- The wider editor layout with grouped fields
- Action buttons aligned at the top

Best screen:

- Edit Ticket view on iPad portrait or landscape

### 3. Reports on iPad

Show:

- A wider reporting screen using iPad space well
- Filters plus summary tables

Best screen:

- Reports page on iPad

### 4. Customers or Products on iPad

Show:

- Table management view with search and action buttons
- A layout that feels clearly tablet-oriented

Best screen:

- Customers or Products page on iPad landscape

### 5. Settings or template tools

Show:

- The advanced admin/configuration side of the app
- Tabs or design tools if they look polished with seeded data

Best screen:

- Settings page with the designer, preview, or print layout tab selected

## Capture order

1. Sign in with the demo account
2. Capture Tickets list
3. Open a ticket and capture Edit Ticket
4. Open preview and capture Ticket Preview
5. Capture Reports
6. Capture Customers or Products
7. Repeat the strongest flows on iPad

## Final review before upload

- Check for clipped safe-area content
- Check that no placeholders or blank fields dominate the image
- Check that totals and labels are readable at App Store sizes
- Check that the icon and launch screen are consistent with the screenshots
